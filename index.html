EVENT FORMAT FOR BLIND CODING

Event Format

All that you need to know about Blind Coding

Blind Coding is a coding competition comprising 2 rounds. Students Have to complete all 2 rounds during the 1-day event. The total score across the 2 rounds will be taken into account for each student. The top three students will be selected at the end of the event.

Round 1: No Lifetime

Duration:

  • This round comprises 3 brief coding questions of an easy level in which you got to code them without utilizing your delete key.

  • This means you are not permitted to redress your past mistakes.

Round 2: Blind Date

Duration:

  • In the last round, the participants will be given 1 real-life problem-solving question.

  • Each member should compose the code for a given issue with their monitor-off.

  • The members are not allowed to see their screens while coding.

Rules:

  • It is mandatory for students to bring their laptops.

  • The entire event consists of 2 rounds with a total of 35 points.

  • Each student will be allowed to participate in all the rounds.

  • Participants whose code gets executed will be given higher preference.

  • The second preference will be given to the code with the least number of errors.

  • Cannot present with plagiarism and published repetitive works

  • In case of the same points at the end of the competition, ties will be broken by the total time for each student in ascending order of time.

  • Any participant found to be indulging in any form of malpractice will be immediately disqualified.

  • Each student is only allowed to choose from the given programming languages to code in.

  • The time limit given will be the same for everyone.

Eligibility:

  • All students (UG and PG) with valid identity cards from their respective educational institutes are eligible to participate.

  • Every participant must have a college ID card. Without it, you cannot participate.

  • On-spot registrations are available.
